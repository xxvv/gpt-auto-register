# Plan: Firefox Extension v2 for GPT Auto Register (Disabled User Friendly)

## Overview
Create `D:\code\gpt-auto-register\firefox-extension-v2` as a new Firefox extension that provides a simple 2-step wizard UI for automating ChatGPT registration and payment link acquisition. Designed for manual-input pain relief: prominent steps at top, country selector + card input in main panel, "Start" button to trigger full automation, and persistent Chinese log output at bottom.

Mimic existing `firefox-extension` for:
- Card parsing format (sidebar.js:1685-1726): split by "----" into 6/7 parts (card, expiry, cvv, [phone|url], name, address).
- Payment link logic (checkoutLinkOnly + requestChatGptCheckoutLinkOnly): use region config (JP/US/ID/DE), call /backend-api/payments/checkout with accessToken.

Support countries: 日本 (JP), 美国 (US), 印度 (ID), 德国 (DE) — map to checkout regions.

## Core Logic (sidebar.js)
- On load: restoreState (country, cardInput), render steps, bind events.
- "开始" click:
  1. Parse card (reuse parseCardInput logic from existing).
  2. Set checkoutRegion from select (JP/US/ID/DE).
  3. Create new tab: https://chatgpt.com
  4. Inject/execute registration automation script via tabs.executeScript (or content script).
  5. Log every major step in Chinese at bottom (e.g. "步骤1: 打开 chatgpt.com", "等待 #email 出现...", "生成邮箱使用域名: minima.edu.kg 等", "提交邮箱", "轮询验证码...", "输入验证码", etc.).
- Registration flow (in injected script or sequential executeScript):
  - Click register button (find by text or selector).
  - Poll every 1s for #email up to 60s → error on timeout.
  - Generate email: localPart (month+day+random) @ one of [minima.edu.kg, watermelon.edu.kg, ciat.edu.kg, cars.edu.kg, damahou.edu.kg] — reuse generateLocalPart + DOMAINS subset.
  - Fill #email, click type=submit.
  - Wait for navigation to https://auth.openai.com/email-verification or /email-verification/register.
  - Handle two paths:
    - Standard: poll name="code", fetch code via CODE_API (reuse fetchVerificationCode logic), fill, submit.
    - Merged (/register): fill code + random name + random age(22-55), submit.
  - Then about-you page: fill name="name" (random Chinese/English name), name="age" (22-55), submit.
  - Reach chatgpt.com → log success.
- Step 2: After registration success, trigger checkoutLinkOnly logic with selected region (reuse requestChatGptCheckoutLinkOnly + regionConfig including DE).
- All errors → log failure + stop.
- Use same helpers: delay, readJsonResponse, CODE_API, showOutput to log div (adapt from elements.payUrlOutput etc.).
- Random name/age generation helpers.
